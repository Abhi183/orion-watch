# Orion-Watch

Orion-Watch is an automated trajectory monitoring pipeline for Artemis II OEM data.
It fetches the latest ephemeris, computes orbital mechanics features in Rust, runs ML
analysis in Python, generates visualizations, and commits generated artifacts.

## Pipeline Stages

1. **Fetch**: downloads latest OEM (`oem-fetch`) with bootstrap fallback.
2. **Rust Engine**: parses OEM + computes engineered orbital features (`oem-engine`).
3. **ML**: runs anomaly detection, forecasting, and landing prediction.
4. **Viz**: renders static mission plots and dashboards.
5. **Commit**: commits updated outputs back to the repository.

## Repository Layout

- `crates/fetcher`: OEM fetch CLI.
- `crates/oem-engine`: OEM parser + feature engine.
- `python/ml/models.py`: ML pipeline.
- `python/viz/plots.py`: visualization generation.
- `.github/workflows/watch.yml`: scheduled and manual GitHub Actions workflow.

## Local Run

```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
cargo test --workspace
cargo build --release --workspace

mkdir -p data outputs
./target/release/oem-fetch --out data/latest.oem --source auto || cp data/bootstrap.oem data/latest.oem
./target/release/oem-engine --input data/latest.oem --features-out data/features.csv --summary-out data/features_summary.json
PYTHONPATH=python python python/ml/models.py data/features.csv data/features_summary.json outputs
PYTHONPATH=python python python/viz/plots.py outputs/features_annotated.csv outputs/ml_results.json
```

## GitHub Actions Reliability Notes

- Uses `actions/checkout@v5` and `actions/setup-python@v6` for Node.js 24 compatibility.
- Enables `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24=true` explicitly.
- Uses `requirements.txt` for deterministic Python dependency installation and cache keying.
- Ensures the ML step reads `data/features_summary.json` generated by the Rust engine.
